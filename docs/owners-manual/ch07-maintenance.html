Updating AECMS

To update to a newer version of AECMS:

  1. Pull the latest changes from the upstream repository into your fork on GitHub.
  2. For Cloud Run: push to the deploy branch — GitHub Actions handles the rest.
  3. For Railway/Render: Railway/Render automatically redeploys when you push to your connected branch.
  4. For Coolify: click Redeploy in the Coolify dashboard.

AECMS runs database migrations automatically on startup (prisma migrate deploy), so schema changes are applied without manual steps.

Backups

Database

  • Railway: Database backups are available in the Railway dashboard under your PostgreSQL plugin.
  • Coolify/VPS: Set up automated pg_dump backups. Store dumps in Backblaze B2 or another off-server location.
  • Cloud SQL: Enable automated backups in the Cloud SQL console. Backups are stored in Google Cloud Storage.

File Storage

  • Local storage: Back up the /app/uploads volume regularly. Consider mirroring to Backblaze B2.
  • Cloud storage (R2/B2/GCS): These services provide redundancy automatically. Enable versioning if you need point-in-time recovery.

Transaction Export

Go to Admin → Orders and use the Export CSV button to download a transaction history. Select a date range (defaults to the current calendar month). The CSV is suitable for import into accounting software (QuickBooks, Wave) or AI-assisted bookkeeping.

Monitoring Costs

For Cloud Run deployments, check GCP Billing monthly. The largest cost item is Cloud SQL (~$7–9/month for the smallest instance). Cloud Run itself is nearly free at low traffic (pay per request).

For Railway/Render/Coolify, your monthly bill is fixed (subscription) rather than usage-based.
