Chapter 5 — Admin Settings & Backstage

A reference for every settings tab and backstage tool.

Settings Tabs

All settings live at /admin/settings. Each tab saves independently — click Save Changes within each section. The Owner can access all tabs; Admins can access Appearance only (by default).

General

Site title, tagline, timezone, date format, homepage mode, registration controls, and CAPTCHA.

Homepage Mode

Controls what visitors see at your root URL (/):

  • Article Feed — redirects / to /articles.
  • Custom Page — shows a specific published page at /. Select the page from the dropdown. If the page is unpublished, the site falls back to _home_ then to the article feed.

Registration Controls

The two registration-control settings live at the bottom of the General tab.

Default Registration Role

The role automatically assigned to new accounts when they register. Defaults to Member. If you create a custom role (e.g. "Patron" with premium content access), you can make it the default here — new sign-ups receive that role automatically.

⚠️ Elevating the default role

If you set a higher-privilege role as the default, consider enabling Require Registration Approval or activating CAPTCHA so that elevated access is not handed to bots or throwaway accounts.

The currently-set default role cannot be deleted until you change this setting. This prevents accidentally breaking the registration flow.

Require Registration Approval

When toggled on, new accounts must be manually approved by an admin before they can log in:

  1. User registers and verifies their email address.
  2. All users who hold the registration.approve capability (Admins and Owners by default) receive an email notification.
  3. Approvers visit /admin/registrations to approve or reject.
  4. Rejection requires a written reason. The account is soft-deleted, freeing the email address for re-registration.
ℹ️ Existing users are not affected

Enabling approval does not lock out already-verified users. The system auto-approves all currently verified accounts the moment you toggle the setting on.

CAPTCHA

AECMS integrates Cloudflare Turnstile for registration-form bot protection. The Managed widget type works silently — real users see at most a brief "Verifying…" indicator, no image puzzles required.

Getting your keys

  1. Log into the Cloudflare Dashboard → Turnstile → Add Site.
  2. Enter your domain. Add localhost for local testing.
  3. Select widget type: Managed.
  4. Copy the Site Key and Secret Key.

Activating CAPTCHA

  1. Go to Settings → General and scroll to the CAPTCHA section.
  2. Paste your Site Key and Secret Key into the fields. The secret is stored encrypted (AES-256-GCM) and never returned by any API.
  3. Click Save Changes. A green "CAPTCHA active" indicator appears when both keys are set.
  4. Test in a private browser window at /auth/register.
✅ No rebuild required

CAPTCHA keys are fetched at runtime from the settings database. You can activate, deactivate, or rotate keys any time — no code push or redeploy needed.

Disabling or rotating keys

Clear either key field and save to disable. To rotate, paste new keys and save — the change takes effect immediately.

Email / SMTP

Enter your SMTP credentials here. Required: Host, Port, Username, Password, From Address.

Click Test to send a test email using the current form values — you do not need to save first. After a successful test, click Save.

Payment Providers

Enter Stripe and PayPal credentials. Click Verify after entering credentials to confirm the API connection before going live.

  • Stripe — handles cards, Apple Pay, Google Pay, and Amazon Pay via Stripe Checkout. Requires a Secret Key and a Webhook Secret (from stripe listen or your dashboard).
  • PayPal — alternative checkout. Requires Client ID and Client Secret. Set mode to sandbox for testing, live for production.
⚠️ Stripe webhook secret

The webhook secret is different from your API key. It begins with whsec_ and is generated either by the Stripe CLI (stripe listen) during development, or from your Stripe Dashboard → Developers → Webhooks in production. Without it, AECMS cannot confirm payment completion from Stripe.

File Storage

Shows the active storage provider and bucket configuration. Fields set via environment variables appear with an env var badge and are read-only in the UI — change them by updating the environment variable and redeploying. You can override env-var values here; the database value takes precedence.

Click Test Storage Connection to verify a write/read/delete round-trip using the current form values.

⚠️ Changing bucket names after upload

If you change a bucket name after files have been uploaded, existing media URLs and download links break. Migrate files to the new bucket before changing the name here.

Appearance

Choose a color palette and font pairing. Changes apply site-wide immediately after saving.

Available palettes: Midnight, Ocean, Forest, Sunset, Rose, Storm, Ember, Arctic.

Font pairings: Classic, Modern, Serif, Mono, Display — each is a heading/body combination.

Custom palettes generated by the Mul Converter also appear here once saved.

Backstage Tools

Users — /admin/users

Lists all registered accounts (excluding soft-deleted users). Requires the user.list capability (Admin and Owner by default).

Searching

The search box filters by email, username, first name, or last name in real time.

Assigning a role

Click the role badge next to a user's name to open a dropdown of all available roles. Select the new role and confirm. Requires the user.assign_role capability (Owner only by default).

ℹ️ You cannot change your own role

The Owner's role is fixed and cannot be changed via the Users page. To promote another user to Owner-level access, assign them the Owner role — but be aware that the Owner role carries all capabilities and cannot be restricted.

Deleting a user

Click the trash icon. Deletion is a soft-delete — the account is hidden from the site and cannot log in, but the record is retained for data integrity (order history, comments, audit log). Requires the account.delete.limited or account.delete.any capability.

  • account.delete.limited — can delete users who do not themselves hold a delete capability. Admins have this by default.
  • account.delete.any — can delete any non-Owner user. Owner-only by default.

Roles & Permissions — /admin/roles

Gated on the role.manage capability (Owner only by default). Defines what each role can do.

Canonical roles

RoleProtectionNotes
OwnerFull — immutableAll capabilities always. Cannot be edited or deleted.
AdminNoneDefault set covers content + ecommerce management. Freely editable.
MemberNoneLogged-in customers — comments, reviews, purchases. Freely editable.
GuestConstrainedApplies to logged-out visitors. Virtual — no user ever holds this role. Only customer-scoped capabilities may be assigned. Cannot be deleted.

Custom roles

Click New Role. Enter a slug (lowercase, hyphens allowed) and a display label. Assign capabilities using the checkbox matrix. Custom roles appear in the role dropdown on the Users page.

Capability scopes

  • Backstage — requires a backstage session (admin dashboard login). Examples: content.publish, order.manage, user.list.
  • Customer — available in the customer-facing experience (standard login). Examples: comment.article, purchase.digital.
⚠️ Deleting a role with active users

A role cannot be deleted while any user holds it. The system reports how many users are affected. Reassign those users first, then retry the deletion.

Media Library — /admin/media

The Media Library manages all uploaded files. It has two tabs:

  • Media Library — images and other public-facing assets (product photos, article hero images, etc.).
  • Digital Files — source files for digital products (PDFs, EPUBs, ZIPs, MP3s, etc.). These are never directly publicly accessible — they are served only via time-limited download tokens after purchase.

Uploading

Drag files onto the upload zone, or click Choose Files. Multiple files can be uploaded at once. Supported image types: JPEG, PNG, GIF, WebP, AVIF, SVG.

Use Bulk Upload for large batches — it shows a progress bar and a per-file success/error report.

Media usage

Each file shows a uses count — the number of content items (articles, products, pages) that reference it. Files with uses are protected from accidental deletion; you must confirm explicitly before deleting them.

Replacing a file

Click Replace on any file to upload a new version. The file record (and all references to it in content) stays intact; only the stored binary changes. Useful for correcting an image without having to re-link it everywhere.

Bulk delete

Select files using the checkboxes and click Delete Selected. If any selected files are in use, the confirmation dialog tells you how many are referenced — you can proceed or cancel.

Navigation Menus

The header navigation is managed from Admin → Pages. Each page has two navigation fields:

  • Show in Nav — toggle to include the page in the header menu.
  • Nav Label — overrides the page title for the nav item (useful for shorter menu labels).
  • Parent Page — nest this page under another to create a submenu.
  • Nav Order — drag-and-drop reorder via the sibling list in the page editor sidebar.

Top-level nav items with children show a dropdown on hover with a configurable delay.

ℹ️ Articles and Shop links

The /articles and /shop links in the header are generated from built-in routes, not from the page system. They are always present and cannot currently be hidden via the nav settings.

Comments & Moderation — /admin/comments

All comments and reviews across articles and products flow through a single moderation queue.

Comment statuses

StatusVisible on site?

ApprovedYes

DeletedNo

AI moderation

When an OpenAI API key is configured, new comments are automatically analyzed for toxicity, spam, and guideline violations before they enter the queue. Comments that pass are auto-approved; borderline ones are flagged for manual review with an AI confidence score. You can override any AI decision.

Reviews vs. comments

A Review is a comment that includes a star rating. Only users with a verified purchase of the product can leave a product review. Article reviews have no purchase requirement. The review score is the average of all approved reviews.

Order confirmation emails

AECMS automatically sends a confirmation email to the customer when their payment is confirmed (Stripe webhook or PayPal capture). The email includes the order number, item list, total, and type-specific information:

  • Digital orders — note that download links are arriving separately.
  • Physical orders — the shipping address on file.
  • Service orders — a note that you will be in contact.

Refunding an order

Open an order and click Refund. You can issue a full or partial refund. The refund is processed directly through the payment gateway (Stripe or PayPal) and the order status updates to Refunded.

Exporting transactions

Click Export CSV on the Orders list to download a transaction history. Select a date range (defaults to the current calendar month). The CSV is suitable for accounting software or AI-assisted bookkeeping.

Registration Queue — /admin/registrations

Visible only when Require Registration Approval is enabled. Shows all accounts that have verified their email and are awaiting manual review.

Click Approve to grant access — the user receives a confirmation email. Click Reject and enter a reason — the account is soft-deleted and the reason is stored in the audit log.

Audit Log — /admin/audit

A chronological record of significant events: logins, content publishes, setting changes, role assignments, order status changes, and more. Each entry records the actor (user ID), the event type, the affected resource, and a metadata blob with before/after values where relevant.

The audit log is append-only and cannot be edited or deleted from the UI. Use it to investigate unexpected changes or security events.

Mul Converter — /admin/mul-converter

Gated on the mul.convert capability (Owner only by default). An AI-powered tool that ingests any public URL and outputs a custom color palette and a page scaffold you can save directly to your site.

Setup

Expand AI Provider Settings at the top of the page. You need at minimum a Text Provider:

  • Anthropic (Claude) — recommended. Sonnet models give the best quality/speed balance.
  • OpenAI (GPT) — alternative.
  • xAI (Grok) — alternative.

Optionally enable an Image Provider to generate placeholder section backgrounds. Supported: GPT-Image-1, xAI Aurora, FLUX (fal.ai), Stability AI.

API keys are stored encrypted in the Internal Secrets Manager. Leave a key field blank on future saves to keep the stored value.

Running an analysis

  1. Enter a URL and click Load Preview.
  2. Toggle Images on or off (requires an image provider).
  3. Click Looks right — Analyze. Allow 30–90 seconds.
  4. Review the palette and page layout. Click any color swatch badge to inspect the CSS value.
  5. Edit palette colors by clicking swatches in the Palette panel.
  6. Choose: Save Palette Only, Create Page Draft, or Save Both.
ℹ️ Preview frame may be blank

Many sites block iframe embedding. A blank preview is expected — the backend fetches the page directly, not through the iframe, so the analysis still works.
