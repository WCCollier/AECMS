Chapter 1 — Choose Your Infrastructure

Four decisions to make before you install anything.

Read time: ~10 minutes

AECMS needs four types of services to run. You can mix and match providers freely. This chapter explains each one and recommends the best option for a low-traffic personal site on a budget.

✅ Make these decisions now

The choices you make here determine which configuration values you will set in Chapter 3. Getting this right upfront takes 10 minutes and prevents you from having to reinstall.

Decision 1 — Hosting Platform

This is where your AECMS application (backend + frontend) actually runs. AECMS runs in Docker containers, so any platform that can run a Docker image will work.

Railway
$5/mo
Easiest setup. Connect GitHub, deploy. PostgreSQL and Redis built in. Best for non-technical owners who want it done.
Coolify + VPS
$5–6/mo
Best value. Run everything on a $5/month server. Free software. More steps to set up, but you own everything and there are no per-service fees.
Google Cloud Run
~$10–12/mo
Most scalable and production-grade. More complex setup. Good if you are already using GCP or want automatic scaling to zero.
Render
~$14+/mo
Similar to Railway. Good UI, managed database. Slightly more expensive. Free tier sleeps after inactivity — not ideal for a live site.
Fly.io
~$0–10/mo
Generous free tier. Requires using the Fly CLI. Better for technically confident owners. Good global edge performance.

Recommendation

If you are not technical and want the simplest possible experience: Railway.
If you want the best price and don't mind a one-time server setup: Coolify on a Hetzner or DigitalOcean VPS.
If you are already comfortable with cloud infrastructure: Google Cloud Run.

Decision 2 — Database (PostgreSQL)

AECMS requires PostgreSQL 15 or later. Most hosting platforms can provide this directly.

Option Free Tier Cost After Notes
Built into Railway Included ~$5–7/mo Simplest. No setup needed — Railway creates it automatically.
Built into Coolify Free (VPS cost only) VPS cost Runs in Docker on same server. Best value by far.
Neon 512 MB, 1 project $19/mo (pro) Scales to zero — first query after idle may be slow. Good free option.
Supabase 500 MB, 2 projects $25/mo ⚠️ Free projects pause after 1 week of inactivity. Bad for a live site.
Cloud SQL (GCP) None ~$7–9/mo Used with Cloud Run. Reliable but not free.
⚠️ Supabase free tier caveat

Supabase is the most marketed free database option, but its free projects pause after 7 days of inactivity. A personal site that goes quiet for a week will become unavailable until you manually unpause it. Use Neon or a paid option instead.

Recommendation

Use whatever your hosting platform provides natively (Railway or Coolify). If you need a standalone managed database, Neon is the best free option (just be aware of the scale-to-zero cold start on the first query after idle). For Cloud Run, use Cloud SQL.

Decision 3 — File Storage

AECMS stores two types of files: public media (images, videos uploaded to articles and products) and private digital product files (PDFs, EPUBs, downloadable files). You need a place to put both.

Option A — Local filesystem (simplest)

Files are stored directly on the server running AECMS. Works out of the box with no extra accounts or configuration. Good for Coolify/VPS deployments where the server has a persistent disk.

  • Cost: $0 (included in VPS/server cost)
  • Downside: Files are tied to the server. You need to handle backups manually. Does not work on platforms with ephemeral containers (Cloud Run).
  • Best for: Coolify, single-server VPS deployments

Option B — Cloudflare R2 (best cloud option)

S3-compatible object storage with zero egress fees. 10 GB free, then $0.015/GB. Works with the S3 storage provider built into AECMS.

  • Cost: Free for 10 GB, then very cheap
  • Setup: Create a Cloudflare account → R2 → Create two buckets → Generate API token
  • Best for: Railway, Render, Fly.io — any platform without persistent disk

Option C — Backblaze B2

10 GB free, then $0.006/GB. S3-compatible. Very popular in self-hosting communities. Egress fees apply (unlike R2), but very cheap at low traffic.

Option D — Google Cloud Storage (GCS)

Required for Cloud Run deployments that use Workload Identity for authentication. The GitHub Actions workflow creates buckets automatically.

Recommendation

  • On Coolify/VPS: start with local storage. Add R2 later if you outgrow the disk.
  • On Railway/Render/Fly: use Cloudflare R2 (zero egress, generous free tier).
  • On Cloud Run: use Google Cloud Storage (automated by the deployment workflow).

Decision 4 — Email / SMTP

AECMS sends transactional emails: account verification, order receipts, password resets, and digital file delivery. You need an SMTP provider to send these.

Provider Free Tier Notes
Resend 3,000 emails/month Developer-friendly, easy setup, excellent deliverability. Recommended.
Brevo (formerly Sendinblue) 300 emails/day Good free tier. Slightly more setup. Good EU alternative.
Mailgun 100 emails/day (trial) Solid deliverability. Requires domain verification.
Gmail SMTP Free Works, but requires an App Password and has sending limits. Not recommended for production.
Your hosting provider Often included Check if your domain registrar or host provides SMTP. Often the simplest option.

Recommendation

Resend is the easiest to set up and has the best deliverability for the free tier. Sign up at resend.com, verify your domain, and copy the API key — that is your SMTP password.

ℹ️ You can configure email after install

Email configuration is done in the Admin Settings panel after the setup wizard runs. You do not need to have it ready before deploying — you can test email delivery later. Just note that verification emails and order receipts will not work until email is configured.

Decision 5 — Redis (Session Cache)

AECMS uses Redis for session token caching, rate limiting, and RSS feed caching. For most platforms, this is handled automatically.

Option Cost Notes
Built into Railway Included Add a Redis plugin to your Railway project. No setup needed.
Built into Coolify Free (VPS cost) One-click Redis service in Coolify. Included in docker-compose.production.yml.
Upstash Free (10K req/day) Serverless Redis. Free tier is sufficient for low-traffic sites. Used with Cloud Run.
Redis Cloud Free (30 MB) 30 MB is very tight. Only if other options aren't available.

Recommendation

Use whatever your hosting platform provides. On Cloud Run, use Upstash (free tier is more than enough for a personal site).
