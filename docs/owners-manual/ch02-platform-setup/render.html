Deploying to Render

Similar to Railway — good UI, managed database.

Cost: ~$14+/month (two web services + database)

ℹ️ Guide coming soon

The full Render setup guide is in progress. In the meantime, Render's setup is similar to Railway:

  1. Fork the AECMS repository to your GitHub account
  2. Create a new Web Service in Render, connected to your GitHub fork
  3. Add a PostgreSQL database via Render's database panel
  4. Add a Redis instance (Render offers managed Redis)
  5. Set environment variables using the values from deploy/render/env.example
  6. Deploy. Render builds from the Dockerfile automatically.
  7. Add your custom domain in Service Settings → Custom Domains

Key difference from Railway: Render's free tier for web services sleeps after 15 minutes of inactivity. Use the paid tier ($7/service/month) for a live site.
