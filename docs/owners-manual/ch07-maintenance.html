Chapter 7 — Maintenance

Backups, updates, password recovery, cost management, and day-to-day operations.

Updating AECMS

To update to a newer version:

  1. Pull the latest changes from the upstream AECMS repository into your fork on GitHub.
  2. Cloud Run: push to the deploy branch — GitHub Actions builds and deploys automatically.
  3. Railway / Render: push to your connected branch — the platform redeploys automatically.
  4. Coolify: click Redeploy in the Coolify dashboard.

Database migrations run automatically on startup (prisma migrate deploy). Schema changes are applied without manual steps.

⚠️ Read the changelog before updating

Some updates include database migrations that are not reversible. Review the CLAUDE.md and phase completion reports before pulling a major update to a live site. Migrations marked "backward compatible" are safe to apply without a maintenance window.

Backups

Database — Neon

If you are on Neon (the recommended database for Cloud Run deployments), backups are handled automatically:

  • Free tier: 7 days of point-in-time restore. Accessible via the Neon dashboard → your project → Restore.
  • Launch tier and above: 30 days of point-in-time restore.

To take a manual snapshot at any time, use pg_dump from a machine with PostgreSQL client tools installed:

pg_dump "postgresql://user:pass@host/neondb?sslmode=require" \
  --no-owner --no-acl -Fc -f backup-$(date +%Y%m%d).dump

Store the dump file in cloud storage (Google Cloud Storage, Backblaze B2, or S3-compatible) so it is off-machine.

Database — Other platforms

  • Railway: database backups are available in the Railway dashboard under your PostgreSQL service.
  • Coolify / VPS: set up automated pg_dump via a cron job. Store dumps off-server (Backblaze B2 or similar).

File Storage

  • Google Cloud Storage, S3, R2, B2: these services provide redundancy automatically. Enable versioning if you need point-in-time file recovery.
  • Local storage (single-server): back up the /app/uploads Docker volume regularly. Consider mirroring to Backblaze B2.

Password Recovery

Users who forget their password can self-serve via /auth/forgot-password. A link on the login page leads there. The flow:

  1. User enters their email address.
  2. If an account exists, a reset link is emailed. The link expires in 1 hour and is single-use.
  3. User clicks the link, sets a new password, and logs in.

The response is always the same success message whether or not the account exists — this prevents user enumeration by third parties.

ℹ️ Owner account recovery

The owner can reset their password via the same flow, provided SMTP is configured. If SMTP is down or the owner's email is inaccessible, recovery requires direct database access — connect via your platform's database console and manually update the password_hash field using bcrypt (cost 12).

Transaction Export

Go to Admin → Orders and click Export CSV to download a transaction history. Select a date range (defaults to the current calendar month). The CSV includes order number, date, customer, items, total, payment method, and status. Suitable for import into QuickBooks, Wave, or AI-assisted bookkeeping tools.

Digital Product Delivery

After a digital product purchase, AECMS automatically:

  1. Creates time-limited download tokens for each purchased file.
  2. Sends a per-file delivery email with the download link.
  3. Sends an order confirmation email with the order summary.

Download tokens expire after a configurable window. If a customer reports their link expired, go to Admin → Orders → the order → Digital Files → Regenerate Link to issue a new token.

Send to Kindle

Customers who purchase a compatible file (PDF or EPUB) can send it directly to their Kindle device from their account page. They need to add their Kindle email address to their account and add your configured sender address to their Amazon approved senders list. Instructions are shown in the Kindle wizard on the account page.

Railway / Render / Coolify

These platforms use flat monthly subscriptions rather than usage-based billing. Your bill is predictable regardless of traffic.

Neon Database — Free Tier Limits

The Neon free tier has two limits to monitor:

  • Storage: 500 MB. Check current usage in the Neon dashboard → your project → Storage. At 400 MB, consider moving to the Launch plan ($19/month, 10 GB) or archiving old content.
  • Compute hours: 191.9 hours/month (auto-suspend after 5 minutes idle). At low traffic this is effectively unlimited — the instance wakes in 1–3 seconds on the first request after idle.
ℹ️ First request after idle

Neon auto-suspends the database after 5 minutes of inactivity on the free tier. The first request after a cold period takes 1–3 seconds while Neon wakes the compute. Subsequent requests within the same session are at full speed. This is acceptable for a low-traffic personal site.

Security Hygiene

  • Rotate the Settings Encryption Key (SETTINGS_ENCRYPTION_KEY) if you suspect it has been exposed. After rotating, re-enter all encrypted credentials in the Settings UI — the old ciphertexts are permanently unreadable under the new key.
  • Review the Audit Log (/admin/audit) periodically for unexpected login events, settings changes, or role modifications.
  • Keep 2FA enabled on the owner account. If you switch authenticator apps, disable and re-enable 2FA in the backstage account settings.
