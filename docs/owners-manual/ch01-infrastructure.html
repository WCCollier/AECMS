Chapter 1 — Choose Your Infrastructure

Five decisions to make before you install anything — and how to make them cheaply.

Read time: ~12 minutes

AECMS needs five types of services to run. You can mix and match providers freely. This chapter explains each one and recommends the best option for a low-traffic personal site on a budget — including how to get very close to $0/month if you choose carefully.

✅ Make these decisions now

The choices you make here determine which configuration values you will set in Chapter 3. Getting this right upfront takes 10 minutes and prevents you from having to reinstall.

Decision 1 — Hosting Platform

This is where your AECMS application (backend + frontend) actually runs. AECMS runs in Docker containers, so any platform that can run a Docker image will work.

Railway
$5/mo base
Easiest setup. Connect GitHub, deploy. PostgreSQL and Redis built in. Best for non-technical owners who want it done fast. No free tier as of 2026 — $5/mo Hobby plan required, then usage-based on top. Expect $10–15/mo with Postgres + Redis.
Coolify + VPS
~€4.50/mo
Best value. Run everything on a Hetzner CX22 (€4.49/mo). Coolify software is free. PostgreSQL, Redis, backend, and frontend all on one server. More steps to set up, but you own everything and there are no per-service fees.
Google Cloud Run
$0–12/mo
Most scalable. Containers scale to zero at idle — you pay only for actual requests. At low traffic, compute is essentially free. The main cost is Cloud SQL (~$10/mo). Can be configured for near-zero cost with Neon Postgres instead. More complex to set up.
Render
~$14+/mo
Similar to Railway. Good UI. Free tier sleeps services after 15 minutes of inactivity (30–50 second cold start) — not suitable for a live site. Minimum usable paid setup is $7/mo per service.
Fly.io
~$5–15/mo
Good global edge performance. Requires the Fly CLI. Free tier is now just a 2-hour trial. For a running app, expect $5–15/mo depending on resources. Better for technically confident owners.

Recommendation

Simplest experience: Railway — one dashboard, no server management, $10–15/mo all-in.
Best value: Coolify on Hetzner — everything for ~€4.50/mo (~$5), one-time setup.
Near-zero for the technically comfortable: Google Cloud Run with Neon Postgres — $0–2/mo at low traffic after free tiers.

ℹ️ How Cloud Run can be near-free

Cloud Run's free tier is generous: 2 million requests/month, 360,000 vCPU-seconds, and 180,000 GiB-seconds of memory — all free, every month, no expiry. A personal site with a few hundred visitors per day will never exceed this. The trick is pairing Cloud Run with Neon Postgres (free tier, 0.5 GB) and Upstash Redis (free tier, 500K commands/month) instead of Cloud SQL. Compute costs: essentially $0. Storage costs a few cents per GB on GCS. This is how the reference deployment at fantasyvreality.com runs for close to nothing in compute.

Decision 2 — Database (PostgreSQL)

AECMS requires PostgreSQL 15 or later. Most hosting platforms can provide this directly.

Option Free Tier Cost After Notes
Built into Railway Included in Hobby plan Usage-based (~$5–10/mo) Simplest. Railway creates it automatically. Counts toward your usage credits.
Built into Coolify Free (VPS cost only) VPS cost Runs in Docker on same server. Best value by far.
Neon ✅ 0.5 GB, genuinely free, no card needed $19/mo (pro) Scales to zero after 5 min idle (first query after idle adds ~1–3s). 10,000 pooled connections via built-in PgBouncer. The best free database option for Cloud Run.
Supabase ⚠️ 500 MB — but pauses after 7 days inactive $25/mo Free projects pause after a week of no traffic. A personal site that goes quiet for a week becomes unavailable. Avoid for production.
Cloud SQL (GCP) ❌ No free tier ~$10/mo minimum db-f1-micro + 10 GB SSD costs ~$7–10/mo. The public IPv4 address alone adds $9.57/mo. Main cost driver of a Cloud Run deployment. Can be replaced with Neon to save ~$10/mo.
⚠️ Supabase free tier pauses projects

Supabase is heavily marketed but its free projects pause after 7 days of inactivity and require a manual unpause. A personal site that goes quiet for a week will become unavailable. Use Neon instead — it also scales to zero, but resumes automatically on the next query (a 1–3 second delay on the first request after idle, then instant).

Recommendation

Use whatever your hosting platform provides natively (Railway or Coolify). If you need a standalone managed database, Neon is the best free option — 0.5 GB is plenty for a personal site's content database (articles, products, users, orders), and the auto-resume cold start is barely noticeable. For Cloud Run, Neon is now the preferred choice over Cloud SQL if you want to minimise cost; see the Cloud Run guide for details.

Decision 3 — File Storage

AECMS stores two types of files: public media (images uploaded to articles and products) and private digital product files (PDFs, EPUBs, etc.). You need a place to put both.

Option A — Local filesystem (simplest)

Files are stored directly on the server running AECMS. Works out of the box with no extra accounts. Good for Coolify/VPS deployments where the server has a persistent disk.

  • Cost: $0 (included in VPS cost)
  • Downside: Files are tied to the server. You must handle backups. Does not work on platforms with ephemeral containers (Cloud Run, Render).
  • Best for: Coolify, single-server VPS deployments

Option B — Cloudflare R2 ★ Recommended for cloud deployments

S3-compatible object storage with permanently zero egress fees. Unlike AWS S3 and Google Cloud Storage, you never pay for data going out to visitors.

  • Free tier: 10 GB storage, 1 million write operations/month, 10 million read operations/month — permanently free, no 12-month limit
  • After free tier: $0.015/GB storage, $4.50 per million writes, $0.36 per million reads — still very cheap
  • Setup: Cloudflare account → R2 → two buckets → API token. Takes about 10 minutes.
  • Reality check: A personal site with 500 uploaded images averaging 500 KB each uses about 250 MB — well within the free 10 GB. You could upload tens of thousands of images before paying a cent.
  • Best for: Railway, Cloud Run, Render, Fly.io — any platform without persistent disk

Option C — Backblaze B2

10 GB free, then $0.006/GB storage. S3-compatible. Very popular in self-hosting communities. Unlike R2, egress fees apply (first 1 GB/day free, then $0.01/GB) — but at low traffic this is negligible.

Option D — Google Cloud Storage (GCS)

Required for Cloud Run deployments using Workload Identity authentication. Two buckets are created automatically by the deployment workflow. First 5 GB free, then $0.020/GB. At a few hundred MB, the cost is pennies per month.

Recommendation

  • On Coolify/VPS: start with local storage. Add R2 if you outgrow the disk.
  • On Railway/Render/Fly: use Cloudflare R2 — zero egress, generous permanent free tier.
  • On Cloud Run: use Google Cloud Storage (automated) — a few cents per month at low traffic.

Decision 4 — Email / SMTP

AECMS sends transactional emails: account verification, order receipts, password resets, and digital file delivery. You need an SMTP provider.

Provider Free Tier Notes
Resend ★ 3,000 emails/month, 100/day Developer-friendly, API key works as SMTP password, excellent deliverability. Easiest setup. Recommended.
Brevo (formerly Sendinblue) 300 emails/day Good free tier. Slightly more setup. Good EU alternative to Resend.
Mailgun 100 emails/day (trial period) Solid deliverability. Requires domain verification. Trial emails expire after 3 months without a paid plan.
Gmail SMTP Free Works, but requires an App Password and has sending limits (500/day). Not recommended for production — deliverability is lower.
Your hosting provider Often included Check if your domain registrar or host provides SMTP. Often the simplest and cheapest option.

Recommendation

Resend is the easiest to set up and has the best deliverability on the free tier. Sign up at resend.com, verify your domain, copy the API key — that is your SMTP password in AECMS settings. 3,000 emails/month is more than enough for a personal site.

ℹ️ Configure email after install

Email is configured in Admin Settings → Email after the setup wizard runs. You can deploy without it and get email working later. Just be aware that verification emails, order receipts, and digital file delivery won't work until it's configured.

Decision 5 — Redis (Cache)

AECMS uses Redis for session token caching, rate limiting, and RSS feed caching. For most platforms this is handled automatically; it is a small workload.

Option Cost Notes
Built into Railway Usage-based (minimal) Add a Redis plugin to your Railway project. No setup needed.
Built into Coolify Free (VPS cost only) One-click Redis service. Included in docker-compose.production.yml.
Upstash ★ ✅ Free: 500K commands/month, 256 MB Serverless Redis. The free tier increased significantly in 2026 — previously 10K/day, now 500K/month. For a personal site making a few thousand cache reads per day, this is effectively free forever. Used with Cloud Run.
Redis Cloud Free (30 MB) 30 MB is tight. Only if other options aren't available.

Recommendation

Use whatever your hosting platform provides. On Cloud Run, use Upstash — the free tier (500K commands/month) comfortably covers a personal site's Redis usage with room to spare.

✅ The near-zero path

Cloud Run + Neon + Upstash + Cloudflare R2 + Resend is the cheapest possible production-grade deployment. All five services have generous free tiers that a personal site will comfortably stay within. The catch: this requires modifying the Cloud Run GitHub Actions workflow to use Neon instead of Cloud SQL (the default). If you are comfortable editing a YAML file and a few environment variables, this is worth it. The Cloud Run guide explains both paths.

ℹ️ Cloud Run free tier specifics (as of 2026)

Google Cloud Run's free tier is per billing account, every month, with no expiry:

  • 2 million requests/month free — a 10,000 page view/month site generates maybe 30,000–50,000 backend requests. You have 40× headroom before paying.
  • 360,000 vCPU-seconds/month free — that is 100 hours of CPU time. Even with fast requests averaging 10ms each, 2 million requests uses only ~5,500 vCPU-seconds.
  • 180,000 GiB-seconds/month free — roughly 200 hours for a 512 MB container.

At low traffic, Cloud Run compute is effectively free. The FvR deployment has run at essentially $0 in compute costs since launch.
