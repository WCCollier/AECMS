Chapter 2 — Platform Setup

Step-by-step setup for your chosen hosting platform.

Select your platform from the guides below. Each guide walks you through:

  1. Creating the platform account and project
  2. Setting up the database and Redis
  3. Deploying the AECMS containers from GitHub
  4. Setting required environment variables
  5. Mapping your custom domain
Railway
$5/mo hobby
Easiest setup. PostgreSQL and Redis plugins built in. Deploy directly from GitHub. Recommended for non-technical owners.
Coolify + VPS
$5–6/mo VPS
Best value. Self-host everything on a cheap VPS. One-time server setup, then a Coolify dashboard handles the rest.
Google Cloud Run
~$10–12/mo
Most scalable. Automated deployment via GitHub Actions. GCS buckets created automatically. Requires a GCP account.
Render
~$14+/mo
Similar to Railway. Good UI. No truly free tier for running services (web services sleep on free plan).
Fly.io
~$0–10/mo
Generous free tier. Requires Fly CLI. Better for owners comfortable with a terminal.
ℹ️ Domain timing

All platforms give you a default URL (e.g. myapp.railway.app) immediately after deploy. You can run the setup wizard on that URL while your custom domain's DNS propagates. See Chapter 6 for domain mapping details.
