Chapter 5 — Admin Settings

A reference for every settings tab in the backstage.

Full guide coming soon — key sections documented below.

General

Site title, tagline, timezone, date format, homepage mode, registration controls, and CAPTCHA. The Homepage setting controls what visitors see at your root URL (/):

  • Article Feed — redirects / to /articles
  • Custom Page — shows a specific page at /. Select a published page from the dropdown. If the selected page is not published, the site falls back to the _home_ page, then to the article feed.

Registration Controls

The two registration-control settings live at the bottom of the General tab.

Default Registration Role

The role automatically assigned to new accounts when they register. Defaults to Member. If you create a custom role (e.g. "Patron" with premium content access), you can make it the default here — new sign-ups will receive that role automatically.

⚠️ Elevating the default role

If you set a higher-privilege role as the default, consider enabling Require Registration Approval (below) or activating CAPTCHA so that elevated access is not handed to bots or throwaway email addresses.

The currently-set default role cannot be deleted until you change this setting to another role. This prevents accidentally breaking the registration flow.

Require Registration Approval

When toggled on, new accounts must be manually approved by an admin before they can log in. The approval workflow:

  1. User registers and verifies their email address.
  2. All users who hold the registration.approve capability (Admins and Owners by default) receive an email notification.
  3. Approvers visit /admin/registrations to approve or reject the account.
  4. Rejection requires a written reason, which is included in the notification email and stored in the audit log. The account is soft-deleted, freeing the email address for re-registration.
ℹ️ Existing users are not affected

Enabling approval does not lock out users who are already verified. The system auto-approves all currently verified accounts the moment you toggle the setting on.

The approval gate is best for tightly controlled communities. For public sites that just need bot protection, CAPTCHA (below) is a lighter-weight alternative — real users are not inconvenienced and no admin action is needed per registration.

CAPTCHA

AECMS integrates Cloudflare Turnstile for registration-form bot protection. Turnstile's Managed widget type works silently in the background — legitimate visitors see at most a brief "Verifying…" indicator, with no image puzzles or checkbox clicks required.

Getting your Turnstile keys

  1. Log into the Cloudflare Dashboard.
  2. In the left nav, go to Turnstile.
  3. Click Add Site.
  4. Enter your domain (e.g. yourdomain.com). For local testing you can add localhost as a second hostname.
  5. Select widget type: Managed (recommended — fully invisible for genuine users).
  6. Copy the Site Key and Secret Key.

Activating CAPTCHA

  1. Go to Settings → General and scroll to the CAPTCHA section near the bottom.
  2. Paste your Site Key into the CAPTCHA Site Key field.
  3. Paste your Secret Key into the CAPTCHA Secret Key field. The secret is stored encrypted in the Internal Secrets Manager (AES-256-GCM) and is never returned in any API response.
  4. Click Save Changes. A green "CAPTCHA active" indicator appears when both fields are populated.
  5. Open the registration form at /auth/register in a private window — the Turnstile widget should appear below the Terms checkbox.
✅ No rebuild required

Unlike typical CAPTCHA integrations that bake the site key at Docker build time, AECMS fetches it at runtime from the settings database. You can activate, deactivate, or rotate CAPTCHA keys any time through the Settings panel — no code push or redeploy needed.

Disabling CAPTCHA

Clear either the Site Key or the Secret Key field in Settings → General and save. The widget will stop appearing on the registration form immediately.

Rotating keys

If you generate new Turnstile keys (e.g. to decommission an old site entry), simply paste the new keys into the settings fields and save. The change takes effect on the next page load.

Email / SMTP

Enter your SMTP credentials here. Required fields: Host, Port, Username, Password, From Address. Click Test to send a test email using the values currently in the form (you do not need to save first). Save after a successful test.

Payment Providers

Stripe and PayPal credentials. Click Verify after entering credentials to confirm the API connection is working. Stripe handles cards, Apple Pay, Google Pay, and Amazon Pay. PayPal is an alternative checkout option.

File Storage

Shows the active storage provider and bucket configuration. Fields set via environment variables appear with an env var badge and are read-only — to change them, update the environment variable and redeploy. You can override env var values by entering a value here; the in-database value takes precedence.

Click Test Storage Connection to verify a write/read/delete round-trip against your storage provider using the values currently in the form.

⚠️ Changing bucket names after files are uploaded

If you change the media or digital bucket name after files have already been uploaded, existing media URLs and download links will break. You must migrate files to the new bucket before changing the name here.

Appearance

Choose a color palette and font pairing. Changes apply site-wide immediately after saving. Available palettes: Midnight, Ocean, Forest, Sunset, Rose, Storm, Ember, Arctic. You can also create and save custom palettes using the Mul Converter or the Design Library (Phase 27).

Roles & Permissions

Found at /admin/roles (gated on the role.manage capability, Owner-only by default). The Role Manager lets you define exactly what each role can do.

Canonical roles

RoleProtectionNotes
OwnerFullAll capabilities always. Cannot be edited or deleted.
AdminNoneDefault set covers content + ecommerce management. Freely editable.
MemberNoneLogged-in customers — comments, reviews, purchases. Freely editable.
GuestConstrainedApplies to logged-out visitors. No user record ever holds this role — it is virtual. Only customer-scoped capabilities may be assigned. Cannot be deleted.

Custom roles

Click New Role to create a role with a slug name (lowercase, hyphens) and a display label. Assign any combination of capabilities using the checkbox matrix. Custom roles appear in the role dropdown on the Users page.

Capability matrix

Capabilities are grouped into Backstage (requires a backstage session — admin dashboard access) and Customer (available in the customer-facing experience). Within each group they are further sorted by category: content, ecommerce, users, system.

⚠️ Deleting a role with assigned users

A role cannot be deleted while any user holds it. The system will report the number of affected users. Reassign those users to another role first, then retry the deletion.

ℹ️ Assigning a role to a user

Role assignment is done on the Users page (/admin/users), not here. The Role Manager is for defining what roles can do; the Users page is for assigning those roles to people.

Mul Converter

Found at /admin/mul-converter (gated on the mul.convert capability, Owner-only by default). The Mul Converter is an AI-powered tool that ingests any public URL and outputs a custom color palette and a page scaffold you can save directly to your site.

Setup

Expand AI Provider Settings at the top of the page. You need at minimum a Text Provider (the AI that analyzes the page and writes the scaffold):

  • Anthropic (Claude) — recommended. Get a key at console.anthropic.com. Sonnet models give the best balance of quality and speed.
  • OpenAI (GPT) — alternative. Get a key at platform.openai.com.
  • xAI (Grok) — alternative.

Optionally enable an Image Provider to have the AI generate placeholder background images for sections that call for them. Supported: GPT-Image-1 (OpenAI), xAI Aurora, FLUX (via fal.ai), Stability AI.

Paste your API key and click Save Settings. Keys are stored encrypted in the Internal Secrets Manager. Leave the key field blank on future saves to keep the stored key.

Running an analysis

  1. Enter the target URL and click Load Preview to confirm the page loads.
  2. Toggle Images on or off (only available if an image provider is configured).
  3. Click Looks right — Analyze. Analysis takes 30–90 seconds; do not close the tab.
  4. Review the results: a color palette and a page layout diagram. The layout badges show background colors/gradients as live swatches — click any badge to see the full CSS value.
  5. Edit any palette color by clicking its swatch in the Palette panel.
  6. Choose a save action: Save Palette Only, Create Page Draft, or Save Both. The page draft opens in the page editor where you can swap placeholder images for real uploads and refine the content.
ℹ️ Site preview may be blank

Many sites block iframe embedding. If the preview frame is blank, that is expected — the analysis still works because the backend fetches the page directly, not through the iframe.
