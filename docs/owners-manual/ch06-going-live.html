Chapter 6 — Going Live

Domain mapping, SSL, and pre-launch checklist.

Domain Mapping by Platform

Each platform handles custom domains slightly differently. The table below shows when and how to map your domain.

PlatformWhen to mapHowSSL
Railway After first deploy Service Settings → Networking → Custom Domain → add CNAME in your DNS Automatic (Let's Encrypt)
Coolify During setup (Step 2) Set A record to VPS IP → Coolify handles domain via Traefik Automatic (Let's Encrypt)
Cloud Run After first deploy gcloud beta run domain-mappings create → add A/AAAA records Automatic (Google-managed)
Render After first deploy Service Settings → Custom Domains → add CNAME in your DNS Automatic (Let's Encrypt)
Fly.io After first deploy fly certs add yourdomain.com → add A/AAAA records Automatic (Let's Encrypt)
✅ Run the wizard before DNS is ready

You do not need to wait for DNS to run the setup wizard. Use the platform's default URL (e.g. *.railway.app) to complete the wizard, then update APP_URL and redeploy once DNS is live.

⚠️ Set APP_URL to your final domain first

APP_URL is baked into authentication cookies and email links at deploy time. Set it to your final custom domain (e.g. https://myblog.com) in your platform's environment variables before deploying — even if DNS isn't active yet. If you change it after setup, you will need to redeploy.

Pre-Launch Checklist

  • ☐ Setup wizard completed — owner account created
  • ☐ Two-factor authentication configured
  • ☐ Email test passed — SMTP credentials saved
  • ☐ Storage test passed — bucket names confirmed
  • ☐ Stripe Verify passed (if taking payments)
  • ☐ Homepage published and set in General Settings
  • ☐ Custom domain mapped and SSL active
  • ☐ APP_URL updated to final custom domain
