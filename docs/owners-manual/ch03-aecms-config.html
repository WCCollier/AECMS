Chapter 3 — Configure AECMS

The environment variables that connect your chosen services to AECMS.

AECMS is configured entirely through environment variables. You set these in your hosting platform's dashboard (Railway, Coolify, Render) or in your GitHub repository's Variables and Secrets (Cloud Run). No files in the repository need to be edited.

ℹ️ Most configuration happens in Admin Settings

Credentials for Stripe, PayPal, and SMTP are configured in the Admin Settings panel after the setup wizard runs. You only need the environment variables below for the initial deployment.

Minimum Required Variables

These must be set before your first deployment:

VariableDescriptionHow to generate
DATABASE_URL PostgreSQL connection string Provided by your database service
REDIS_URL Redis connection string Provided by your Redis service
JWT_SECRET Signs authentication tokens openssl rand -hex 64
SETTINGS_ENCRYPTION_KEY Encrypts credentials stored in Admin Settings openssl rand -hex 32 — save this
APP_URL Your site's full public URL e.g. https://myblog.com
NODE_ENV Runtime environment Always production

Storage Variables

VariableDefaultDescription
STORAGE_PROVIDER_TYPElocallocal, gcs, or s3
STORAGE_PATH/app/uploadsLocal storage path (local provider only)
S3_ENDPOINT—S3-compatible endpoint URL (e.g. Cloudflare R2)
S3_REGIONus-east-1Storage region
GCS_BUCKET_MEDIA—GCS media bucket name (Cloud Run only)
GCS_BUCKET_DIGITAL—GCS digital bucket name (Cloud Run only)

Bucket names and S3 credentials can also be configured in Admin Settings → File Storage after install.

Email Variable

VariableValue
EMAIL_PROVIDER_TYPEsmtp (for production) or console (dev mode — no emails sent)

SMTP credentials (host, port, username, password, from address) are configured in Admin Settings → Email after install.

Platform-Specific Files

Template env files for each platform are in the deploy/ directory of the repository:

  • deploy/railway/env.example — Railway variables
  • deploy/coolify/env.example — Coolify / VPS variables
  • deploy/cloud-run/env.example — GitHub Variables and Secrets for Cloud Run

For a complete list of all available variables, see deploy/env.reference.md.
