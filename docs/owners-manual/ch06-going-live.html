Chapter 6 — Going Live

Domain mapping, SSL, and pre-launch checklist.

Domain Mapping by Platform

PlatformWhen to mapHowSSL
Railway After first deploy Service Settings → Networking → Custom Domain → add CNAME in DNS Automatic (Let's Encrypt)
Coolify During setup Set A record to VPS IP → Coolify handles domain via Traefik Automatic (Let's Encrypt)
Cloud Run After first deploy gcloud beta run domain-mappings create → add A/AAAA records Automatic (Google-managed)
Render After first deploy Service Settings → Custom Domains → add CNAME in DNS Automatic (Let's Encrypt)
Fly.io After first deploy fly certs add yourdomain.com → add A/AAAA records Automatic (Let's Encrypt)
✅ Run the wizard before DNS is ready

You do not need to wait for DNS propagation to run the setup wizard. Use the platform's default URL to complete the wizard, then update APP_URL and redeploy once your domain is active.

⚠️ Set APP_URL to your final domain before the wizard

APP_URL is used in authentication cookies and all outbound email links. Set it to your final custom domain (e.g. https://myblog.com) in your platform's environment variables before running the wizard — even if DNS isn't active yet. Changing it later requires a redeploy and may break existing email links.

Pre-Launch Checklist

Complete all items before announcing your site publicly.

Core Setup

  • ☐ Setup wizard completed — owner account created
  • ☐ Two-factor authentication configured on the owner account
  • ☐ Custom domain mapped and SSL active
  • ☐ APP_URL set to your final custom domain and redeployed
  • ☐ Homepage published and set in Settings → General

Email

  • ☐ SMTP credentials saved in Settings → Email
  • ☐ Test email sent and received successfully
  • ☐ Test the forgot-password flow: go to /auth/login → "Forgot password?" — confirm reset email arrives and the link works

Payments (if taking payments)

  • ☐ Stripe credentials saved and Verify passed
  • ☐ Stripe webhook secret configured (whsec_... from dashboard or stripe listen)
  • ☐ PayPal credentials saved and Verify passed (if offering PayPal)
  • ☐ End-to-end test purchase completed — confirm order confirmation email arrives

File Storage

  • ☐ Storage provider configured and Test Connection passed
  • ☐ At least one media file uploaded successfully

Security

  • ☐ CAPTCHA configured — Turnstile keys saved in Settings → General (recommended for any public registration form)
  • ☐ Registration controls reviewed — set Default Role and decide whether to enable Approval Gate

Legal

  • ☐ Terms of Service page reviewed, edited, and published (auto-generated at wizard completion — requires your review before publishing)
  • ☐ Privacy Policy page reviewed, edited, and published
  • ☐ Both pages linked in your site footer or navigation
⚠️ Auto-generated policies are a starting point only

The Terms of Service and Privacy Policy pages generated during setup are a generic template. They are created in draft status and must be reviewed by you — ideally with input from a qualified legal professional — before you publish them. Laws vary by jurisdiction and business type.

Content

  • ☐ At least one published article or page (the site should not be empty when announced)
  • ☐ About page or author bio published
  • ☐ Any sample/placeholder content removed or moved to draft
