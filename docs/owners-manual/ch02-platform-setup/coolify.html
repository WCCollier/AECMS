Deploying with Coolify on a VPS

Best value: everything on one €4.49/month server, managed through a Heroku-like dashboard.

Estimated time: 45–60 minutes  ·  Cost: ~€4.49/mo (Hetzner CX22) — no per-service fees

✅ What you get for €4.49/month

Coolify is free, open-source software that turns a cheap VPS into a Heroku-like platform. For the cost of one coffee per month, a Hetzner CX22 (2 vCPU, 4 GB RAM, 40 GB disk, 20 TB monthly traffic) runs the entire AECMS stack: PostgreSQL, Redis, NestJS backend, and Next.js frontend — all in Docker containers, managed through a GUI, with automatic SSL via Let's Encrypt. There are no per-service fees, no usage charges, and no cold starts.

Note on Hetzner pricing: Hetzner raised prices in April 2026 by roughly 20–30%. The CX22 is now €4.49/mo (was €3.29/mo). It remains by far the best value VPS in Europe for this workload.

Before You Start

  • A GitHub account with the AECMS repository forked to your account
  • An account with a VPS provider (recommended: Hetzner, DigitalOcean, or Linode/Akamai)
  • A domain name with access to its DNS settings

Step 1 — Create a VPS

  1. Log in to your VPS provider and create a new server (called a "Droplet" on DigitalOcean, a "Linode" on Linode, or a "Server" on Hetzner).
  2. Choose these settings:
    • OS: Ubuntu 24.04 LTS
    • Size: 2 GB RAM / 2 vCPU minimum (this is the $6/month tier on most providers)
    • Region: closest to where your visitors are located
  3. Add your SSH key if prompted, or note the root password provided.
  4. Create the server and note its IP address.

Step 2 — Point Your Domain at the Server

Do this now, while Coolify installs, so DNS has time to propagate.

  1. In your domain registrar's DNS settings, find the A record for your root domain (@ or blank).
  2. Change its value to your VPS's IP address.
  3. Also add an A record for www pointing to the same IP.
  4. If you use DNS TTL, set it to 300 (5 minutes) to speed up propagation.

DNS changes can take up to 24 hours to propagate worldwide, but usually take 15–30 minutes.

Step 3 — Install Coolify

  1. SSH into your server: 
    ssh root@YOUR_SERVER_IP
  2. Run the Coolify installer: 
    curl -fsSL https://cdn.coollabs.io/coolify/install.sh | bash
    This installs Docker and Coolify. It takes about 2–3 minutes.
  3. Open Coolify in your browser at http://YOUR_SERVER_IP:8000. Create your Coolify admin account when prompted.
ℹ️ Coolify runs on port 8000

Your AECMS site will eventually run on ports 80 and 443 (standard HTTP/HTTPS). Coolify itself runs on port 8000. Coolify manages Traefik, a reverse proxy that handles routing and SSL for your apps.

Step 4 — Add Your Server to Coolify

  1. In Coolify, go to Servers → Add Server.
  2. Select Localhost (the server Coolify is already installed on). Coolify will validate it automatically.

Step 5 — Create a PostgreSQL Database

  1. Go to Databases → New Database → PostgreSQL.
  2. Set a name (e.g. aecms-postgres), choose your server, set a strong password.
  3. Click Start Database. Coolify starts a PostgreSQL container and shows you a connection string.
  4. Copy the Internal Connection String — it starts with postgresql://. You'll need this for AECMS.

Step 6 — Create a Redis Instance

  1. Go to Databases → New Database → Redis.
  2. Set a name and password, choose your server, then Start Database.
  3. Copy the Internal Connection String (starts with redis://).

Step 7 — Deploy AECMS from GitHub

  1. Go to Projects → New Project → give it a name like aecms.
  2. Click New Resource → Application → GitHub.
  3. Connect your GitHub account and select your forked AECMS repository.
  4. Set the Base Directory to /backend and the Dockerfile location to Dockerfile. Click Continue.
  5. Set the Port to 4000.
  6. Under Domains, add your API subdomain, e.g. api.yourdomain.com — or use the same domain as the frontend with Coolify's path-based routing.
  7. Save and repeat for the frontend: create another Application, set Base Directory to /frontend, Port to 3000, Domain to yourdomain.com.

Step 8 — Set Environment Variables

In Coolify, click your backend application → Environment Variables. Add:

NODE_ENV=production
DATABASE_URL=postgresql://aecms:YOUR_DB_PASS@YOUR_DB_HOST:5432/aecms
REDIS_URL=redis://default:YOUR_REDIS_PASS@YOUR_REDIS_HOST:6379

JWT_SECRET=YOUR_64_CHAR_RANDOM_STRING
SETTINGS_ENCRYPTION_KEY=YOUR_32_CHAR_HEX_KEY
JWT_EXPIRATION=15m
REFRESH_TOKEN_EXPIRATION=7d
SETTINGS_KMS_PROVIDER=local

STORAGE_PROVIDER_TYPE=local
STORAGE_PATH=/app/uploads

APP_URL=https://yourdomain.com
FRONTEND_URL=https://yourdomain.com
API_URL=https://yourdomain.com
FRONTEND_ADMIN_URL=https://yourdomain.com/admin
# Email — configure in Admin Settings → Email after launch

For the frontend application, add:

NODE_ENV=production
BACKEND_URL=http://aecms-backend:4000
NEXT_PUBLIC_API_URL=https://yourdomain.com
NEXT_PUBLIC_BASE_DOMAIN=yourdomain.com
⚠️ Internal vs external URLs

Within Coolify, containers communicate using internal Docker network names (e.g. http://aecms-backend:4000). The NEXT_PUBLIC_API_URL is the public URL that browsers use — it should be your domain.

Step 9 — Deploy

  1. Click Deploy on the backend application. Watch the build logs.
  2. Once the backend is healthy, deploy the frontend.
  3. Coolify automatically provisions SSL certificates via Let's Encrypt once your domain's DNS has propagated.
  4. Visit https://yourdomain.com — you should see the AECMS site or the setup wizard.

Step 10 — Add a Persistent Volume for Uploads

If you are using local file storage, you need a persistent volume so uploaded files survive container restarts.

  1. In Coolify, go to your backend application → Storage.
  2. Add a volume: source /data/aecms/uploads (on the server), destination /app/uploads (in the container).
  3. Redeploy the backend.

Domain Setup — Already Done

Because you pointed your domain to the server's IP in Step 2, and Coolify uses Traefik with automatic SSL, your domain mapping is handled automatically once DNS propagates. No additional configuration needed.

✅ Best time to run the setup wizard

You can run the wizard as soon as the backend is deployed and healthy. Use the server's IP address (http://YOUR_IP:3000) if your domain's DNS hasn't propagated yet.
